# Roadmap

This repository is documentation-first and focused on helping users run Claude Code through FreeLLMAPI and LiteLLM with a clear, reproducible setup.

## Current Priorities

- Keep the setup instructions aligned with the final verified scripts and config examples.
- Preserve valid internal links across root docs, `docs/`, `guides/`, and `examples/`.
- Keep placeholder values explicit so no secret-like tokens or real credentials appear in examples.
- Keep the docs-validation workflow lightweight and documentation-only.

## Near-Term Work

- Re-run the documentation validation workflow after meaningful doc changes.
- Verify the startup scripts whenever configuration examples change.
- Keep the landing page concise so readers understand the repo immediately.

## Longer-Term Ideas

- Add a small compatibility matrix for the most common local setups.
- Expand troubleshooting notes with more real-world failure modes.
- Add a maintainer checklist for release-time documentation review.
